SkipList简介

前言

SkipList是一种集快速查找，插入，删除等功能于一身的内存数据结构,广泛应用于RocksDB，LevelDB等数据库的内存部分。通过学习RocksDB的机会，我了解到了这种以前从未听说过的数据结构。本文将简要介绍关于SkipList的一些基本知识，后续可能还会加入一些实现和优化

概述

多层次的 有序 链表。最底层是一个包含了所有元素的有序链表，高层则是由较低层次的子序列形成的有序链表。高层次的链表相比于低层次的链表跳过了一些元素，所以被称为跳表。一个简单的跳表示意图如下所示：

跳表的查询效率可以达到$O(\log n)$, 最一点足以与平衡二叉树媲美，但skiplist的实现复杂度低于平衡二叉树。同时，由于平衡二叉树存在大量的平衡-旋转操作，当大量的数据插入或修改时，会对平衡二叉树的性能造成巨大的影响，甚至导致阻塞。

[1] 的作者提出了一种独特的视角来理解skiplist: 将skiplist去掉一些重复边之后旋转，可以将skiplist看作是一棵二叉树：

上面的右图并不算是严格的Binary Search Tree, 因为左子树的数据并不全是小于根节点的数据。但查询过程有一些类似，当我们想要查询30时，我们发现30大于7，因此到7的右子树里面查找，然后发现30小于53，所以到25的左子树中查找；进一步发现25小于42，再到左子树中查找，最后找到30。如下图所示：

从二分查找到SkipList

[2]中也提出了一种有趣的思考SkipList的方式，就是从二分查找的角度来思考。举例来说, 如果我们有一个有序的数组：1，3，19，23，57，62。那么要从里面查找某个元素，例如23，通过二分查找来是非常高效的。但是对于同样排列的一个有序链表而言，二分查找却不能生效。原因在于链表并不支持随机的下标访问。通过双指针法倒是能够在每次获取到中间位置的节点，但是遍历仍需$O(n)$的代价，总共需要$O(n\log n)$的代价。

虽然无法直接通过“下标”的方法获取得到中间位置的节点，但我们可以将该节点保存下来，将其作为最高层，然后对于左半部分，获取其中间节点的位置并保存下来；右半部分也做同样的操作，形成第二层；依次进行下去，最后得到完整的“树”型结构。这就是跳表的雏形：

当然这并不是跳表的最后形式，在跳表中还有一些随机化的因素，所以最后的跳表并不一定完全遵循上面的形式，同时还存在一些可能的优化。比方说，假设我们要查询7这个节点，那么我们先查询整个表的“中间节点”， 这可以通过读取最高层的元素得到；我们发现是4，7 > 4, 所以向右半部分查找。此时我们需要查找右半部分的中间位置，这个值怎么得到呢？按图中来看，我们需要从第二层开始向右移动3个位置才能得到6这个节点，这个移动本身可能是非常耗时的，尤其是当查询达到最底层时，需要移动到该链表非常靠后的位置时，这个最坏情况需要$O(n)$ 的复杂度。

一个解决办法就是在两层的相同元素之间添加自上而下的指针。例如图中最高层的4到第二层的4需要添加一个指针，这样当我们需要获取右半边的中间元素时，就可以使用：4->4->6这样的序列而不是2->4->6这样的序列。

一些术语规定

包含了所有数据的有序链表称为最低层，往上高度越高。

一个数据的 height 是指整个skiplist中包含该数据的节点所在的最高的高度。例如在上面包含7,11,22,25这副图中，11的高度是1, 15的高度是2，25的高度是3。

插入节点

除非特别说明，否则我们都认为插入节点时该数据在SkipList中没有出现。为了插入一个节点，我们需要对这个新插入的节点分配一个高度，然后从该高度开始向下的每一层都插入相同数据的节点。

新插入节点的高度由如下概率决定：$p(n)=2^{1-n}$， 其中n表示高度。该分布意味着高度越高，节点处在这个高度的概率越小。

就插入方法而言，[1]中使用了递归的插入方法，[2]中使用的则是求出每层中要插入节点的后继的方法然后进行插入。但总地来说，插入方法还是比较单一的：通常使用自顶向下的插入方法，在插入节点的高度位置，先找到该节点的前驱和后继进行插入，然后通过前驱的向下指针索引到下一层的对应位置，从该位置开始向后查找前驱和后继，如此循环，直到最后在最底层完成插入。

删除

删除和插入类似，但是删除的简单之处在于只要在该节点的高度位置找到该节点，就能通过向下指针直接删除跳表中的所有相同元素。

查找

SkipList的查找很简单，采用自顶向下的查找方法。从最高层开始，假设要查找的数据是k, 找到链表中的第一个位置n, 使得n <= k < n+1, 如果k等于n，则直接使用向下指针一路索引到最底层即可；否则n < k, k介于n和n+1在下一层的节点之间。通过向下指针到达下一层继续开始查找，直到最后到达最底层为止。如下图所示：
